# MD-01 â€” Environment-Based Configuration (No Hardcoding)

## Purpose
Remove all hardcoded localhost URLs and make the system deployable
via environment variables without changing runtime behavior.

## In Scope
- Replace frontend hardcoded backend URL with env-based config
- Replace backend hardcoded CORS origin with env-based config
- Ensure local dev still works with defaults
- No feature changes

## Out of Scope
- No deployment scripts
- No infra changes
- No data loading changes

## Requirements

### Frontend
- Use `NEXT_PUBLIC_API_URL`
- Provide a safe default for local development
- No direct `localhost` references in code

### Backend
- Use `CORS_ORIGINS` (comma-separated)
- Default to `http://localhost:3000` if unset
- Must not break local dev

## Success Criteria
- App works locally without env vars
- App works when env vars are provided
- No hardcoded localhost URLs remain
